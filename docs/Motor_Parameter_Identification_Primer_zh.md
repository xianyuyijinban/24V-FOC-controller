# 电机参数识别算法入门手册（基于本项目）

## 1. 这份手册适合谁
这份文档给第一次接触电机参数识别和FOC整定的同学使用。目标是让你理解本项目现有识别流程在做什么、为什么这样做、怎么判断结果是否可信。

## 2. 为什么要识别参数
FOC控制依赖电机参数。参数不准会直接带来电流环震荡、速度环发抖、发热偏大、力矩不线性等问题。

本项目参数结构在 `MotorParam_t`，核心字段如下。

- `Rs`：定子电阻
- `Ld/Lq`：电感
- `Ke`：反电动势常数
- `Pn`：极对数
- `J/B`：惯量与粘性阻尼
- `theta_offset`：编码器零位偏移

对应代码位置：
- `MDK-ARM/code/motor_identify.h`
- `MDK-ARM/code/motor_identify.c`
- `MDK-ARM/code/foc_app.c`

## 3. 本项目识别状态机总览
识别主状态机在 `MI_Process()`，顺序如下。

1. `MI_STATE_PN_IDENTIFY`
2. `MI_STATE_RS_IDENTIFY`
3. `MI_STATE_LS_IDENTIFY`
4. `MI_STATE_KE_IDENTIFY`
5. `MI_STATE_J_IDENTIFY`
6. `MI_STATE_ENCODER_ALIGN`
7. `MI_STATE_COMPLETE` 或 `MI_STATE_ERROR`

状态机入口：
- `FOC_App_StartIdentify()`
- `FOC_App_TIM1_IRQHandler()` 中周期调用 `MI_Process()`

说明：
- 识别过程已经绑定到 TIM1 控制周期执行，和PWM/采样同步。
- 识别完成后保存到Flash，供下次上电直接加载。

## 4. 各参数识别原理（按项目实现）

### 4.1 极对数 Pn 识别
函数：`MI_IdentifyPn()`

实现思路：
- 给定开环旋转电角度 `theta_elec_cmd`
- 固定 `Iq` 产生拖动
- 读取编码器机械角变化
- 用电角总变化和机械角总变化比值估算 `Pn`

核心公式：
- `Pn ≈ |Δtheta_elec / Δtheta_mech|`

你要关注：
- 机械角累计量太小会导致识别失败
- 编码器噪声或打滑会导致 `Pn` 偏差

### 4.2 定子电阻 Rs 识别
函数：`MI_IdentifyRs()`

实现思路：
- 在 alpha 轴注入直流电压，先正向再反向
- 等待电流稳定后采样
- 分别计算 `Rs_positive`、`Rs_negative`
- 取均值并做收敛性检查

核心公式：
- `Rs = V / I`

正反向都测的目的：
- 抵消死区、电压偏置等方向性误差

### 4.3 电感 Ld/Lq 识别
函数：`MI_IdentifyLs()`

实现思路（当前版本）：
- 注入已知频率、已知幅值正弦电压
- 统计电流 RMS
- 先算阻抗 `Z = Vrms / Irms`
- 再扣除电阻分量得到感抗 `XL`
- 最后 `L = XL / (2*pi*f)`

核心公式：
- `Vrms = Vamp / sqrt(2)`
- `Z = Vrms / Irms`
- `XL = sqrt(max(Z^2 - Rs^2, 0))`
- `L = XL / (2*pi*f)`

实现假设：
- `Ld = Lq`（各向同性简化）

### 4.4 反电动势常数 Ke 识别
函数：`MI_IdentifyKe()`

实现思路：
- 先开环加速到目标电角速度
- 再进入测量窗口，使用编码器差分得到实测 `omega_e`
- 在静止坐标系估算反电势幅值 `|Eαβ| ≈ |Vαβ - Rs * Iαβ|`
- 用 `|Eαβ|` 与 `|omega_e|` 的比值估算 `Ke`

核心公式：
- `E_alpha = V_alpha - Rs * I_alpha`
- `E_beta = V_beta - Rs * I_beta`
- `E_mag = sqrt(E_alpha^2 + E_beta^2)`
- `Ke ≈ E_mag_avg / |omega_e_avg|`

注意：
- 这是工程近似方法，精度依赖稳态质量和负载情况

### 4.5 惯量 J 与阻尼 B
函数：`MI_IdentifyJ()`

当前项目状态：
- 这里是简化实现，返回默认值，不是实际在线辨识

建议：
- 上位机调参阶段把 `J/B` 当“可整定参数”而不是“已准确识别参数”

### 4.6 编码器零位偏移 theta_offset
函数：`MI_EncoderAlign()`

当前项目状态：
- 采用锁轴对齐：识别阶段注入 `Id` 锁轴，在已知电角 `0` 下读取机械角
- 通过 `theta_offset = normalize(-theta_mech * Pn)` 计算零位偏移并写入参数

## 5. 识别过程与控制链路关系
识别不是独立系统，它跑在控制主链路中：

1. ADC采样（TIM触发 + DMA）
2. TIM1中断进入控制周期
3. 识别状态机更新激励命令
4. FOC或直接SVPWM更新占空比
5. 下一周期继续采样与计算

这意味着识别质量同时受这些因素影响：
- 电流采样零点是否校准
- 编码器通信稳定性
- 母线电压波动
- 驱动器故障/限流状态

## 6. 怎么判断识别结果是否可信

### 6.1 数值范围初筛
可先用下面经验范围筛查（24V小功率伺服）：

- `Rs`：0.05 ~ 5 ohm
- `L`：几十 uH 到几 mH
- `Ke`：0.005 ~ 0.5 V/(rad/s)
- `Pn`：1 ~ 50（且应是整数）

### 6.2 一致性检查
- 同一台电机重复识别，多次结果应接近
- `Rs` 热态应高于冷态，趋势应符合物理规律
- 带入参数后，电流环响应应明显改善

### 6.3 控制表现验证
- 空载小电流运行是否平稳
- 速度阶跃是否有明显震荡
- 同等扭矩下电流是否异常偏大

## 7. 常见失败原因与排查

1. 编码器抖动或CRC错误多
- 先看 `TLE5012` 数据有效标志和错误计数

2. 电流零点没校准好
- 先做静止校准，再进识别

3. 功率级没真正使能
- 检查 `DRV_EN`、门极驱动状态、PWM输出

4. 电机有外部负载或机械摩擦过大
- 识别期间尽量空载

5. 参数识别窗口太短
- 可适当增加测试时长，提高平均效果

## 8. 你现在应该知道的边界
本项目当前识别是“可用的工程版本”，不是“高精度实验室辨识器”。其定位是：

- 快速得到可运行参数
- 支撑FOC闭环进入稳定工作区
- 给后续人工整定提供起点

不应期待：
- 一次识别直接得到全温区最优参数
- 在重载、强扰动下仍保持同样精度

## 9. 进阶建议（后续可做）

1. 提升零位对齐鲁棒性（例如增加多次采样平均、方向一致性校验）
2. 给 `J/B` 增加加减速辨识法
3. 给 `Ke` 增加多点速度拟合
4. 给 `Ld/Lq` 增加 d/q 轴分离识别
5. 增加识别结果质量评分与自动重试
